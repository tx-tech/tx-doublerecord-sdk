# 项目版本控制规范



## 一、提交日志

### 格式
```
{action} {description}
```

- {action}
    - \+ add
    - \* update or bugfix
    - \- remove
- {description}
    - 尽量描述的清楚简单
    

### 举例
- \+ &nbsp;add event for webview 
- \* &nbsp;update navbar style
- \- &nbsp;remove umeng



## 二、分支命名

### 格式
```
{action}-{shortName}
```

- {action}
    - feature
    - bugfix

### 举例
- feature-umeng
- feature-umeng
- bugfix-navbar
